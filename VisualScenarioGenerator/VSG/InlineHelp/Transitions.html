Introduction to Transitions

A Transition is used by a Singleton Vulnerability to describe how much of a particular capability must be applied to invoke a desired state change.  

A transition has the following parameters:

The capability to which the unit is vulnerable to.

The Intensity or amount of the capability that must be applied.

And two optional parameters; range and probability.  Range says the contribution must be applied within this distance.  Probability is the probability that if the assets are within range that the unit will be vulnerable.

   

How to Define a Transition

1. Right click on the "Transitions" entry on the Object Types tree to the left.

2. Select Create Transition.  Enter an Transition name in the resulting dialog.

3. Follow the onscreen instructions for defining a transition.
