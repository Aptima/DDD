Introduction to Decision Makers

A Decision Maker is a scenario entity that owns and controls "Units".  A "unit" is a controllable onscreen asset, such as a plane or tank.   In general a Decision Maker is a human player, however, this need not always be the case.

A Decision Maker has five basic components:

  1. A Role, a brief description of the Decision Maker's purpose.
  2. An Identifier, a unique name for the Decision Maker. 
  3. A Color denoting ownership.  All onscreen assets owned by a Decision Maker will appear in the same color.
  4. A collection of communication and control management options which determine who a Decision Maker can communicate with, and who they report to.
  5. A Briefing that lays out the Decision Maker's mission or objective.

How to Define a Decision Maker

  1. Right click on the "Decision Makers" entry on the Object Types tree to the left.
  2. Select Create Decision Maker.  Enter a Decision Maker name in the resulting dialog.
  3. Follow the onscreen instructions for defining a Decision Maker.

Notes About Communication and Control Options

  1. Reports To - This selector allows you to choose a single Decision Maker that this Decision Maker 'reports' to. If the Can Force Asset Transfers checkbox is selected, then this Decision Maker's superior is allowed to transfer this Decision Maker's assets.
  2. Can Share Whiteboard With - This selector defines which other Decision Makers this DM is able to be in a whiteboard room with. Both DMs need to be able to be in the same whiteboard room in order for the room to be created.
  3. Can Chat With - This selector defines which other Decision Makers this DM is able to be in a text chat room with. Both DMs need to be able to be in the same text chat room in order for the room to be created.
  4. Can Speak With - This selector defines which other Decision Makers this DM is able to be in a voice channel with. Both DMs need to be able to be in the same voice channel in order for the channel to be created.

Additionally, you can determine whether or not a decision maker is able to transfer its assets during the simulation.

  • Can Transfer Assets - When checked, this Decision Maker is able to transfer their assets to other DMs in a simulation.
  • Can Force Asset Transfers - When checked, this Decision Maker is able to transfer assets which belong to Decision Makers that Report To this DM.
